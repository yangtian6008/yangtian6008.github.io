webpack 配置文件升级模式

  • 在文件夹下创建webpack专属配置文件webpack.config.js 
                                const path=require('path')      //引入pnm中的初始化模块中的路径方法
                            module.exports={                //ES6语法中的模块化方法将相关属性作为API接口方式向外部，抛出
                            entry:"./src/main.js",           //告诉webpack 打包时的入手文件位置
                            output:{                           //安排打包好后的文件存放路劲和保存的文件名称
                            path:path.resolve(__dirname,'dist'),  //__dirname 是node下的内置参数
                            filename:'bundles.js'}
                                }                               //目的就是为了不用每次都敲打webpack entry_filepath output_filepath  save_filename

webpack 命令的映射配置

  • 经过npm init 初始化后的webpack有一个专属配置文件package.json文件
  •             {
  "name": "firstwebpack",
  "version": "1.0.0",
  "description": "",
  "main": "webpack.config.js",
  "dependencies": {
    "webpack-cli": "^3.3.12",
    "webpack": "^4.44.2"
  },
  "devDependencies": {},
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1",
    "build": "webpack"
  },
  "author": "",
  "license": "ISC"
}
        
  •  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1",
    "build": "webpack"
  • 在我们使用npm run PROJECT 时 实际上就使用运行了上述代码中的script脚本，所以想让webpack映射到pnm窗口只需要在脚本中定义一个字段"Name":"webpack",以后那个Name就可以替代npm run PROJECT中的PROJECT
